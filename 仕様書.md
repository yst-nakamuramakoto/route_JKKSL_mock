# 営業支援ルート最適化アプリ 仕様書（概要レベル）

## 1. 目的

Google Workspace Frontline 導入を前提に、社員のGoogleカレンダーを起点として、営業訪問・既存利用者モニタリングの計画から実行までを支援する。

## 2. 対象機能（4機能）

### 機能1：空きスケジュールへのタスク提案（＋スケジューラー機能）
- **目的**：Googleカレンダーの予定をホーム画面に表示・追加できるようにし、空き時間に対して訪問すべき先（新規営業／既存利用者モニタリング）をAIが提案する。あわせて、Googleカレンダーにはないタスク管理（チェックリスト）機能を提供する。
- **入力データ**：
  - Googleカレンダーの予定・空き時間（Calendar API）
  - 基幹貸与システムの顧客データ（利用状況、前回訪問日、優先度）
  - アプリ内で管理するタスク（期限・優先度・完了状態）
- **出力**：
  - 本日の予定タイムライン表示（カレンダーの予定＋空き時間）
  - ホーム画面からの予定追加（Calendar APIへの書き込み＝Googleカレンダー側にも反映）
  - 提案された訪問先・訪問理由・推奨時間帯のリスト。カレンダーへの自動挿入（任意）。
  - タスク一覧（カレンダー化されていない「やることリスト」。例：見積書送付、電話フォロー等）
- **画面要素**：
  - 本日のスケジュール（タイムライン、予定カード＋空き時間カード）
  - 「＋予定を追加」ボタン（タイトル・時間・場所を入力するフォーム）
  - AI訪問提案カード（顧客名・理由・優先度・所要時間、採用／却下ボタン）
  - タスク管理リスト（チェックボックスで完了、期限・優先度タグ、＋タスク追加）

### 機能2：AI×Googleマップ 最適ルート提案
- **目的**：複数訪問先が決まった際、移動効率の良い巡回順をAIが提案する。
- **入力データ**：訪問先リスト（緯度経度）、出発地、各訪問の希望時間帯。
- **出力**：最適化された訪問順序、地図上のルート表示、移動時間・距離の合計。
- **画面要素**：地図表示、訪問順リスト（ドラッグで手動調整可）、合計移動時間表示。

### 機能3：滞在時間アラート
- **目的**：訪問先での滞在が長引いた場合に通知し、スケジュール遅延を防ぐ。
- **入力データ**：訪問先到着時刻（GPS連動）、カレンダー上の予定終了時刻。
- **出力**：残り時間表示、退出推奨タイミングのプッシュ通知。
- **画面要素**：現在訪問中カードに残り時間カウントダウン、警告色表示（残り10分で黄色、過ぎたら赤）。

### 機能4：駐車場空き状況連携
- **目的**：訪問先周辺の駐車場（タイムズ・リパーク等）の空き状況をルートに組み込む。
- **入力データ**：訪問先周辺の駐車場API（満空情報）。
- **出力**：訪問先ごとの推奨駐車場（空車・距離・料金目安）、ルートへの経由地追加。
- **画面要素**：訪問先カードに駐車場候補を表示（空車／満車アイコン、距離）。

## 3. 画面構成（モック対象）

1. ホーム（本日のスケジュール一覧＋予定追加＋AI訪問提案＋タスク管理：機能1）
2. ルート提案画面（機能2：地図＋訪問順）
3. 訪問中画面（機能3：滞在時間アラート）
4. 駐車場候補画面（機能4：駐車場空き状況）

## 4. 今回のモックの位置づけ

- 見た目・画面遷移・操作感の確認が目的。
- データはすべてダミー（ハードコード）。
- バックエンド・API実連携は行わない（フェーズ2以降で実装）。
