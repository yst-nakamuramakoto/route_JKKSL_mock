# 営業支援ルート最適化アプリ 仕様書（概要レベル）

## 1. 目的

Google Workspace Frontline 導入を前提に、社員のGoogleカレンダーを起点として、営業訪問・既存利用者モニタリングの計画から実行までを支援する。

## 2. 対象機能（4機能）

### 機能1：空きスケジュールへのタスク提案
- **目的**：カレンダーの空き時間に対し、訪問すべき先（新規営業／既存利用者モニタリング）をAIが提案する。
- **入力データ**：
  - Googleカレンダーの空き時間（Calendar API）
  - 基幹貸与システムの顧客データ（利用状況、前回訪問日、優先度）
- **出力**：提案された訪問先・訪問理由・推奨時間帯のリスト。カレンダーへの自動挿入（任意）。
- **画面要素**：空き時間一覧、提案カード（顧客名・理由・優先度・所要時間）、採用／却下ボタン。

### 機能2：AI×Googleマップ 最適ルート提案
- **目的**：複数訪問先が決まった際、移動効率の良い巡回順をAIが提案する。
- **入力データ**：訪問先リスト（緯度経度）、出発地、各訪問の希望時間帯。
- **出力**：最適化された訪問順序、地図上のルート表示、移動時間・距離の合計。
- **画面要素**：地図表示、訪問順リスト（ドラッグで手動調整可）、合計移動時間表示。

### 機能3：滞在時間アラート
- **目的**：訪問先での滞在が長引いた場合に通知し、スケジュール遅延を防ぐ。
- **入力データ**：訪問先到着時刻（GPS連動）、カレンダー上の予定終了時刻。
- **出力**：残り時間表示、退出推奨タイミングのプッシュ通知。
- **画面要素**：現在訪問中カードに残り時間カウントダウン、警告色表示（残り10分で黄色、過ぎたら赤）。

### 機能4：駐車場空き状況連携
- **目的**：訪問先周辺の駐車場（タイムズ・リパーク等）の空き状況をルートに組み込む。
- **入力データ**：訪問先周辺の駐車場API（満空情報）。
- **出力**：訪問先ごとの推奨駐車場（空車・距離・料金目安）、ルートへの経由地追加。
- **画面要素**：訪問先カードに駐車場候補を表示（空車／満車アイコン、距離）。

## 3. 画面構成（モック対象）

1. ホーム（本日のスケジュール一覧：機能1の提案を含む）
2. ルート提案画面（機能2：地図＋訪問順）
3. 訪問中画面（機能3：滞在時間アラート）
4. 駐車場候補画面（機能4：駐車場空き状況）
